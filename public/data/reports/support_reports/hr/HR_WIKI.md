# HR Data Wiki

## 1. Overview
This document serves as the **Single Source of Truth** for the HR Data Layer. It defines the physical tables available in the system and how they map to the "Logical Data Keys" requested by the Business Support Analytics Engine.

## 2. Logical to Physical Mapping

| Logical Key (Report Requirement) | Physical Table ID | Description |
| :--- | :--- | :--- |
| **EMPLOYEES**, `Headcount` | `HR_Employees` | Employee master data. |
| **PAYROLL**, `Salaries` | `HR_Payroll` | Compensation records. |
| **RECRUITMENT**, `Hiring` | `HR_Recruitment` | Job openings and candidates. |
| **TRAINING**, `Learning` | `HR_Training` | Training completion logs. |
| **PERFORMANCE**, `Reviews` | `HR_Performance` | Performance appraisals. |

## 3. Table Definitions

### 3.1 Workforce

#### `HR_Employees`
*Employee master.*
- **employee_id** (`string`, PK)
- **full_name** (`string`)
- **department** (`string`)
- **status** (`string`): `Active`, `Terminated`, `On Leave`.

### 3.2 Talent Management

#### `HR_Recruitment`
*Hiring pipeline.*
- **job_id** (`string`, PK)
- **job_title** (`string`)
- **status** (`string`): `Open`, `Filled`, `Cancelled`.

#### `HR_Performance`
*Performance reviews.*
- **review_id** (`string`, PK)
- **employee_id** (`string`, FK)
- **rating** (`number`): 1-5 scale.
