# Maintenance Data Wiki

## 1. Overview
This document serves as the **Single Source of Truth** for the Maintenance Data Layer. It defines the physical tables available in the system and how they map to the "Logical Data Keys" requested by the Operations Analytics Engine.

## 2. Logical to Physical Mapping

| Logical Key (Report Requirement) | Physical Table ID | Description |
| :--- | :--- | :--- |
| **WORK_ORDERS**, `Repairs` | `MNT_Work_Orders` | Maintenance work orders. |
| **ASSETS**, `Equipment` | `MNT_Assets` | Machinery and assets. |
| **PM_SCHEDULE**, `Preventive` | `MNT_Preventive_Maintenance` | Scheduled maintenance. |
| **TECHNICIANS**, `Labor` | `MNT_Technicians` | Maintenance staff. |
| **SPARE_PARTS**, `Inventory` | `MNT_Spare_Parts` | Spare parts stock. |

## 3. Table Definitions

### 3.1 Work Management

#### `MNT_Work_Orders`
*Corrective and preventive work orders.*
- **work_order_id** (`string`, PK)
- **asset_id** (`string`, FK)
- **status** (`string`): `Open`, `In Progress`, `Closed`.
- **priority** (`string`): `Low`, `Medium`, `High`, `Critical`.

### 3.2 Asset Management

#### `MNT_Assets`
*Master list of equipment.*
- **asset_id** (`string`, PK)
- **asset_name** (`string`)
- **criticality** (`string`): `A`, `B`, `C`.

#### `MNT_Preventive_Maintenance`
*PM Schedules.*
- **plan_id** (`string`, PK)
- **frequency** (`string`): `Weekly`, `Monthly`, `Annual`.
- **next_due** (`date`)
