# Sales Data Wiki

## 1. Overview
This document serves as the **Single Source of Truth** for the Sales Data Layer. It defines the physical tables available in the system and how they map to the "Logical Data Keys" requested by the Business Analytics Engine.

## 2. Logical to Physical Mapping

| Logical Key (Report Requirement) | Physical Table ID | Description |
| :--- | :--- | :--- |
| **SALES_DATA**, `Revenue` | `SLS_Orders` | Sales transactions. |
| **CUSTOMERS**, `Accounts` | `SLS_Customers` | Customer master data. |
| **PIPELINE**, `Opportunities` | `SLS_Opportunities` | CRM opportunities. |
| **CONTRACTS**, `Agreements` | `SLS_Contracts` | Sales contracts. |

## 3. Table Definitions

### 3.1 Transactions

#### `SLS_Orders`
*Sales orders.*
- **order_id** (`string`, PK)
- **customer_id** (`string`, FK)
- **total_amount** (`number`)
- **status** (`string`)

### 3.2 CRM

#### `SLS_Opportunities`
*Sales pipeline.*
- **opportunity_id** (`string`, PK)
- **stage** (`string`): `Prospecting`, `Negotiation`, `Closed Won`.
- **estimated_value** (`number`)
