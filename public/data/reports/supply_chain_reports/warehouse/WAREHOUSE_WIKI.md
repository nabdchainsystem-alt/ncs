# Warehouse Data Wiki

## 1. Overview
This document serves as the **Single Source of Truth** for the Warehouse Data Layer. It defines the physical tables available in the system and how they map to the "Logical Data Keys" requested by the Supply Chain Analytics Engine.

## 2. Logical to Physical Mapping

| Logical Key (Report Requirement) | Physical Table ID | Description |
| :--- | :--- | :--- |
| **INVENTORY_DATA**, `Stock` | `WH_Inventory_Balance` | Current stock levels. |
| **WAREHOUSE_OPS**, `Movements` | `WH_Inventory_Transactions` | Inbound/Outbound movements. |
| **RECEIPTS**, `Inbound` | `WH_Inbound_Receipts` | Goods receipt logs. |
| **PICKING_DATA**, `Outbound` | `WH_Picking_Tasks` | Picking task performance. |
| **LABOR_LOGS**, `Timesheets` | `WH_Labor_Timesheets` | Staff hours and shifts. |
| **EQUIPMENT_LOGS**, `Forklifts` | `WH_Equipment` | Equipment status and maintenance. |
| **SAFETY_INCIDENTS**, `OSHA` | `WH_Safety_Incidents` | Safety records. |
| **ENERGY_DATA**, `Utilities` | `WH_Energy_Consumption` | Energy usage logs. |

## 3. Table Definitions

### 3.1 Inventory

#### `WH_Inventory_Balance`
*Current stock snapshot.*
- **warehouse_id** (`string`, FK)
- **item_id** (`string`, FK)
- **qty_on_hand** (`number`)
- **qty_available** (`number`)

#### `WH_Inventory_Transactions`
*Stock movement history.*
- **transaction_id** (`string`, PK)
- **transaction_type** (`string`): `Receipt`, `Issue`, `Transfer`.
- **qty** (`number`)
- **location_from_id` / `location_to_id` (`string`)

### 3.2 Operations

#### `WH_Picking_Tasks`
*Outbound picking performance.*
- **task_id** (`string`, PK)
- **picker_user_id** (`string`, FK)
- **qty_to_pick** (`number`)
- **completed_at** (`datetime`)

#### `WH_Safety_Incidents`
*Safety logs.*
- **incident_id** (`string`, PK)
- **incident_date** (`date`)
- **severity** (`string`): `Low`, `Medium`, `High`.
- **description** (`string`)
