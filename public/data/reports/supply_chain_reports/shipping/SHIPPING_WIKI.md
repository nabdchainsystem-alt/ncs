# Shipping Data Wiki

## 1. Overview
This document serves as the **Single Source of Truth** for the Shipping Data Layer. It defines the physical tables available in the system and how they map to the "Logical Data Keys" requested by the Supply Chain Analytics Engine.

## 2. Logical to Physical Mapping

| Logical Key (Report Requirement) | Physical Table ID | Description |
| :--- | :--- | :--- |
| **LOGISTICS_DATA**, `Shipments` | `SHP_Shipments` | Master shipment records. |
| **CARRIERS**, `Transport` | `SHP_Carriers` | Carrier profiles. |
| **ROUTES**, `Lanes` | `SHP_Lanes` | Trade lanes and routes. |
| **TRACKING**, `GPS` | `SHP_Tracking_Events` | Real-time tracking. |
| **SHIPPING_COST**, `Freight` | `SHP_Costs` | Freight costs and invoices. |
| **CUSTOMS**, `Trade` | `SHP_Customs_Declarations` | Customs data. |
| **RATES**, `Tariffs` | `SHP_Freight_Rates` | Contracted freight rates. |

## 3. Table Definitions

### 3.1 Operations

#### `SHP_Shipments`
*Master shipment records.*
- **shipment_id** (`string`, PK)
- **shipment_no** (`string`)
- **origin_country** / **destination_country** (`string`)
- **mode** (`string`): `Air`, `Ocean`, `Road`.
- **status** (`string`): `Booked`, `In Transit`, `Delivered`.

#### `SHP_Tracking_Events`
*Real-time tracking updates.*
- **event_id** (`string`, PK)
- **shipment_id** (`string`, FK)
- **timestamp** (`datetime`)
- **event_type** (`string`): `Departure`, `Arrival`, `Delay`.
- **location** (`string`)

### 3.2 Financials

#### `SHP_Costs`
*Freight costs.*
- **cost_id** (`string`, PK)
- **shipment_id** (`string`, FK)
- **amount** (`number`)
- **cost_type** (`string`): `Freight`, `Fuel`, `Duty`.

#### `SHP_Freight_Rates`
*Contracted rate cards.*
- **rate_id** (`string`, PK)
- **carrier_id** (`string`, FK)
- **lane_id** (`string`, FK)
- **base_rate** (`number`)
